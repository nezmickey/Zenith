# Zenith Language Specification

Zenith はポーランド記法（前置記法）・遅延評価を備えた純粋関数型言語である。
この言語の核心は、**「型情報が文法（構造）を決定する」** という設計にあり、括弧による優先順位指定を原則不要とする。

---

## 1. プログラムの構造

プログラムは **文（statement）** の列で構成される。
文は「型シグネチャ宣言」と「関数定義」の2種類のみ存在し、いずれもセミコロン `;` で終端する。

```
<program>   ::= <statement>*
<statement> ::= <type_decl> | <func_def>
```

### 1.1 型シグネチャ宣言

関数（または定数）の名前と型を宣言する。定義の前に必ず宣言しなければならない。

```
<type_decl> ::= <name> "::" <type> ";"
```

**例:**
```
add :: int -> int -> int;
main :: int;
twice :: (int -> int) -> int -> int;
```

### 1.2 関数定義

宣言された関数の実装を与える。
対応する型シグネチャが先に存在していなければならない。

```
<func_def> ::= <name> <param>* "=" <expr> ";"
```

- `<param>` は仮引数名（0個以上）。引数の数はシグネチャから推論される。
- `<expr>` は関数の本体。

**例:**
```
add x y = + x y;
main = add 1 2;
mul2 = * 2;
```

### 1.3 定義順序

- 型シグネチャは、対応する関数定義より先に記述しなければならない。
- 関数間の参照に定義順序の制約はない（相互再帰が可能）。

### 1.4 エントリーポイント

プログラムの実行は `main` という名前の定義から開始される。
`main` は引数を取らない値（例: `int` 型）でなければならない。

---

## 2. 型システム

Zenith は **単相型（monomorphic）** の静的型付け言語である。
すべてのシンボルは一意の型を持ち、型はパース時に完全に決定される。

### 2.1 基本型

| 型名   | 説明                |
| :--- | :---------------- |
| `int`  | 整数型（符号付き整数）       |
| `bool` | 真偽値型（`true` / `false`） |

### 2.2 関数型

```
<type> ::= <base_type>
         | <type> "->" <type>
         | "(" <type> ")"
```

- `->` は **右結合** である。`int -> int -> int` は `int -> (int -> int)` と等しい。
- 括弧 `()` は関数型を「ひとまとまりの型」として扱うために使用する。
  主に高階関数の引数として関数型を渡す場合に必要。

**例:**
```
int                           -- 整数
int -> int                    -- int を受け取り int を返す関数
int -> int -> int             -- カリー化された 2 引数関数
(int -> int) -> int -> int    -- 「int -> int の関数」を第 1 引数に取る高階関数
```

---

## 3. 式（Expression）

### 3.1 リテラル

| 種類 | 構文 | 型 |
| :--- | :--- | :--- |
| 整数リテラル | `0`, `1`, `42`, `256` ... | `int` |
| 真リテラル | `true` | `bool` |
| 偽リテラル | `false` | `bool` |

整数リテラルは `0` 以上の10進数の数字列で構成される。負のリテラルは存在しないが、`- 0 n` で表現可能。

### 3.2 識別子

英字またはアンダースコア `_` で始まり、英数字またはアンダースコアからなる文字列。

```
<identifier> ::= [a-zA-Z_][a-zA-Z0-9_]*
```

### 3.3 関数適用

Zenith は **ポーランド記法（前置記法）** を採用する。
関数名を先に書き、その後に引数を空白区切りで並べる。

```
+ 1 2            -- 1 + 2  → 3
* 3 + 1 2        -- 3 * (1 + 2)  → 9
```

括弧による優先順位指定は原則不要である。代わりに **型駆動の最短一致** アルゴリズムにより結合が決定される。

---

## 4. 型駆動の結合メカニズム

Zenith の最も重要な規則であり、言語の全体構造を支配する。

### 4.1 基本ルール

式のパースは、左からトークンを読みながら以下のルールに従って行われる。

1. **基本要素の読み取り**: 最初のトークン（リテラル、識別子）を読み、その型を確認する。
2. **型の照合**: 読み取ったトークンの型が関数型 (`A -> B`) であり、かつ式に期待される型と一致しない場合、次のトークンを引数として消費する。
3. **最短一致（即時結合）**: 引数に期待される型と次のトークンの型が一致すれば、即座にそのトークンを引数として飲み込む。
4. **待機と右側再帰評価**: 引数に期待される型と次のトークンの型が一致しない場合（例: `int` を求めているのに次が関数型）、右側を再帰的にパースし、結果の型が一致するまで評価させる。
5. **停止条件**: 式の型が期待される型と一致したら、それ以上の引数適用を行わず停止する。

### 4.2 具体例

`+ + 1 2 3` の結合過程（全体の期待型は `int`）:

```
+ :: int -> int -> int    期待: int   型が不一致 → 引数を求める
  引数1の期待型: int
    + :: int -> int -> int    期待: int   型が不一致 → 引数を求める
      引数1の期待型: int
        1 :: int              一致 → 引数1 = 1
      引数2の期待型: int
        2 :: int              一致 → 引数2 = 2
    → (+ 1 2) :: int          一致 → 外側の+の引数1 = 3
  引数2の期待型: int
    3 :: int                  一致 → 引数2 = 3
→ (+ (+ 1 2) 3) :: int       最終結果 = 6
```

### 4.3 高階関数と最短一致

高階関数が関数型を引数に求める場合、右隣の識別子の型が完全に一致すれば、その識別子を **評価せず関数そのもの** として渡す。

```
twice :: (int -> int) -> int -> int;
twice f x = f f x;
add2 :: int -> int;
add2 = + 2;
main :: int;
main = twice add2 10;
```

`twice add2 10` のパースにおいて:
- `twice` は `(int -> int)` 型の引数を期待する。
- `add2` の型は `int -> int` であり、期待型 `(int -> int)` と完全に一致する。
- よって `add2` は評価されず、関数そのものとして `twice` に渡される。

## 5. カリー化と部分適用

### 5.1 カリー化

すべての関数はカリー化されている。
「n 個の引数を取る関数」は存在せず、「1 つの引数を取り、残りの引数を受け取る新しい関数を返す関数」の連鎖として扱われる。

```
add :: int -> int -> int;
add x y = + x y;
```

`add` は `int` を1つ受け取り、`int -> int` 型の関数を返す。
返された関数がもう1つの `int` を受け取り、最終結果の `int` を返す。

### 5.2 部分適用

引数の一部だけを供給して新しい関数を生成できる。

```
mul2 :: int -> int;
mul2 = * 2;
```

ここで `* 2` は `* :: int -> int -> int` に引数 `2` だけを適用し、
「受け取った数を2倍にする関数」(`int -> int`) を生成する。

---

## 6. 遅延評価

Zenith は **遅延評価（Lazy Evaluation）** を採用する。

- 関数に渡される引数は、渡された時点では評価されない。
- 引数の値が実際に必要になった時点（プリミティブ演算や最終結果の出力時）で初めて評価される。
- 一度評価された結果はメモ化され、同じ引数が再度参照されても再計算されない。

**効果の例:**
```
first :: int -> int -> int;
first x y = x;
main :: int;
main = first 1 / 2 0;
```

`/ 2 0`（ゼロ除算）は `first` の第2引数だが、`first` は第1引数 `x` のみを返す。
遅延評価により `/ 2 0` は評価されないため、ゼロ除算エラーは発生せず `1` が返る。

## 7. 組み込みプリミティブ

以下のプリミティブは型宣言なしでそのまま使用できる。

### 7.1 算術演算子

| 演算子 | 型 | 説明 |
| :--- | :--- | :--- |
| `+` | `int -> int -> int` | 加算 |
| `-` | `int -> int -> int` | 減算 |
| `*` | `int -> int -> int` | 乗算 |
| `/` | `int -> int -> int` | 整数除算 |
| `%` | `int -> int -> int` | 剰余 |

### 7.2 比較演算子

| 演算子 | 型 | 説明 |
| :--- | :--- | :--- |
| `==` | `int -> int -> bool` | 等値 |
| `!=` | `int -> int -> bool` | 非等値 |
| `<`  | `int -> int -> bool` | 小なり |
| `>`  | `int -> int -> bool` | 大なり |
| `<=` | `int -> int -> bool` | 以下 |
| `>=` | `int -> int -> bool` | 以上 |

### 7.3 条件演算子（三項演算子）

| 名前 | 型 | 説明 |
| :--- | :--- | :--- |
| `if` | `bool -> int -> int -> int` | 第1引数が `true` なら第2引数を、`false` なら第3引数を返す |

`if` は関数であるため、他のすべての式と同じ型駆動の結合ルールに従う。
条件分岐のネストも括弧なしで記述可能。

```
fibonacci n =
    if == n 1 1
    if == n 2 1
    + fibonacci - n 1 fibonacci - n 2;
```

上記の `if` のネストは以下と等価:
- `if (== n 1) 1 (if (== n 2) 1 (+ (fibonacci (- n 1)) (fibonacci (- n 2))))`

## 8. 字句規則

### 8.1 トークン一覧

| トークン | 構文 | 役割 |
| :--- | :--- | :--- |
| 識別子 | `[a-zA-Z_][a-zA-Z0-9_]*` | 変数名・関数名 |
| 整数リテラル | `[0-9]+` | 数値 |
| 真偽値リテラル | `true`, `false` | 真偽値 |
| 演算子 | `+`, `-`, `*`, `/`, `%`, `<`, `>`, `==`, `!=`, `<=`, `>=` | プリミティブ演算（識別子として扱う） |
| `::` | 型シグネチャ宣言の区切り | |
| `->` | 関数型の区切り | |
| `=` | 定義の区切り | |
| `(` `)` | 型のグルーピング | |
| `;` | 文の終端 | |

### 8.2 空白と改行

空白・タブ・改行はトークンの区切りとしてのみ機能し、意味的な違いはない。
インデントは自由であり、文法上の意味を持たない。

### 8.3 コメント

コメント構文は存在しない。

### 8.4 予約語

なし。`if`, `true`, `false`, `int`, `bool` は文法上の予約語ではなく、組み込み定数・型名として扱われる。

## 9. 文法の形式的定義

```
program     = statement*

statement   = type_decl | func_def

type_decl   = IDENT "::" type ";"

func_def    = IDENT IDENT* "=" expr ";"

type        = base_type
            | type "->" type        (右結合)
            | "(" type ")"

base_type   = "int" | "bool"

expr        = term
            | expr term             (型駆動の最短一致で結合)

term        = NUMBER
            | "true"
            | "false"
            | IDENT

NUMBER      = [0-9]+
IDENT       = [a-zA-Z_][a-zA-Z0-9_]*
            | "+" | "-" | "*" | "/" | "%"
            | "==" | "!=" | "<" | ">" | "<=" | ">="
            | "if"
```

## 10. プログラム例

### 10.1 階乗 (Recursion & If)

再帰呼び出しと `if` 演算子を組み合わせた基本的な例。

```
factorial :: int -> int;
factorial n = if == n 1 1 * n factorial - n 1;
main :: int;
main = factorial 10;
```

**解説:**
- `if == n 1 1 ...` : ポーランド記法により `if (n == 1) 1 (...)` と解釈される。
- `* n factorial - n 1` : 型駆動結合により `n * (factorial (n - 1))` と解釈される。`factorial` は `int -> int` を期待するため、右側の `- n 1` が先に評価される。

### 10.2 フィボナッチ数列 (Multiple Recursion)

複数の再帰呼び出しを含む例。

```
fibonacci :: int -> int;
fibonacci n =
    if == n 1 1
    if == n 2 1
    + fibonacci - n 1 fibonacci - n 2;
main :: int;
main = fibonacci 10;
```

**解説:**
- `if` の入れ子: 第3引数（else部分）に次の `if` を置くことで、`else if` のような構造を括弧なしで実現している。
- `+ f1 f2` : `+` 演算子が2つの再帰結果を待ち受けて加算する。

### 10.3 繰り返し適用 (Higher-Order Function)

関数を引数として取る高階関数の例。

```
repeat :: (int -> int) -> int -> int -> int;
repeat f n x =
    if == n 0
    x
    repeat f - n 1 f x;
mul2 :: int -> int;
mul2 = * 2;
main :: int;
main = repeat mul2 10 1;
```

**解説:**
- `(int -> int)` : シグネチャにおいて括弧を使用し、第1引数が関数であることを明示している。
- `repeat f - n 1 f x` : `repeat` の第1引数に `f` をそのまま渡し、第3引数には `f x` (fをxに適用した結果) を渡している。
- `mul2 = * 2` : 部分適用により「2倍する関数」を生成し、`repeat` に渡している。

### 10.4 高速累乗（O(log n)） (Complex Expressions)

複雑な式の結合例。

```
pow :: int -> int -> int;
pow x n =
    if == n 0
    1
    if == % n 2 1
    * x pow * x x / n 2
    pow * x x / n 2;
main :: int;
main = pow 5 10;
```

**解説:**
- `% n 2` : 剰余演算。`==` は `int` を2つ取るため、`% n 2` と `1` が引数となる。
- `* x pow * x x / n 2` : `x * (pow (x * x) (n / 2))` と解釈される。Zenithの型駆動ルールにより、複雑な数式も括弧なしで一意に定まる。

### 10.5 遅延評価の検証 (Lazy Evaluation)

評価されない枝がエラーを引き起こさないことを示す例。

```
first :: int -> int -> int;
first x y = x;
main :: int;
main = first 1 / 2 0;
```

**解説:**
- `first 1 / 2 0` : `first` は `int -> int -> int` なので、`1` と `/ 2 0` を引数に取る。
- `/ 2 0` はゼロ除算だが、`first` の定義内で `y`（第2引数）が参照されないため、評価（force）が走らず、エラーにならずに `1` を返す。
